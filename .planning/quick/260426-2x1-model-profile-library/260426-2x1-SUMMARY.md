# Quick Task 260426-2x1: Model Profile Library Summary

Model profile library built and fully wired into both containers — stop sequences now flow from architecture detection at startup through every litellm.acompletion() call site.

## What Was Built

### Task 1 (pre-done): model_profiles.py in both containers

Both `modules/pathfinder/app/model_profiles.py` and `sentinel-core/app/services/model_profiles.py` provide:

- `ModelProfile` dataclass: family, stop_sequences, context_window, supports_system_prompt, chat_template_format
- `FAMILY_PROFILES` dict keyed by LM Studio arch strings (qwen2, llama3, llama, mistral, gemma2, phi3, deepseek2) plus aliases
- `SAFE_DEFAULT` — no stop sequences, 4096 ctx, used when arch is unknown
- `get_profile(model_id, api_base)` — async, 3-step resolution: LM Studio /api/v0/models/{id} → substring pattern match → SAFE_DEFAULT; cached per (model_id, api_base)

### Task 2: Pathfinder stop sequences

**resolve_model.py** — `resolve_model_profile(task_kind: TaskKind)` signature fixed (was taking `model_id: str`). Now calls `resolve_model()` internally, strips `openai/` prefix, and calls `get_profile()`.

**llm.py** — Added:
- `_stop_for(profile) -> list[str] | None` helper — returns sequences or None if empty
- `profile: ModelProfile | None = None` parameter on all 9 `acompletion` functions: `extract_npc_fields`, `generate_npc_reply`, `generate_mj_description`, `update_npc_fields`, `generate_harvest_fallback`, `classify_rule_topic`, `generate_ruling_from_passages`, `generate_session_recap`, `generate_story_so_far`, `generate_ruling_fallback`
- `stop=_stop_for(profile)` injected into kwargs at each call site when non-empty

**foundry.py** — Added `_stop_for` helper + `profile` param to `generate_foundry_narrative`.

**Route files wired:**
- `routes/rule.py` — resolves `profile_chat` and `profile_structured` alongside models; passes to `classify_rule_topic`, `generate_ruling_from_passages`, `generate_ruling_fallback`
- `routes/npc.py` — passes `profile=await resolve_model_profile("structured")` to `extract_npc_fields` and `update_npc_fields`; `profile=await resolve_model_profile("fast")` to `generate_mj_description`; resolves `profile_chat` once for `generate_npc_reply` loop
- `routes/harvest.py` — resolves `profile_chat` once; passes to `generate_harvest_fallback`
- `routes/session.py` — resolves `profile_chat` once in each handler; passes to `generate_story_so_far` and `generate_session_recap`
- `routes/foundry.py` — calls `get_profile(bare_model, api_base)` directly (foundry uses a configured model string, not a task kind)

### Task 3: Sentinel-core stop sequences

**litellm_provider.py** — `LiteLLMProvider.complete(messages, stop=None)` — stop sequences injected into kwargs when provided.

**provider_router.py** — `ProviderRouter.complete(messages, stop=None)` — threads stop to primary; fallback intentionally receives no stop (cloud providers manage their own termination).

**main.py lifespan** — After context window discovery, calls `get_profile(_lmstudio_model_name, api_base)`, stores `app.state.lmstudio_stop_sequences`. Non-fatal: logs warning and stores `[]` on any failure.

**routes/message.py** — Reads `app.state.lmstudio_stop_sequences` and passes as `stop=` to `ai_provider.complete()`.
