# Requirements: Sentinel of Mnemosyne

**Defined:** 2026-04-10
**Core Value:** A message goes in, an AI response that knows your history comes back — and what mattered gets written to Obsidian so the next conversation starts smarter.

## v1 Requirements

### Core Infrastructure

- [x] **CORE-01**: Pi harness container starts and accepts HTTP POST requests via a Fastify bridge server wrapping the stdin/stdout JSONL subprocess
  - *Outcome: Delivered v0.1. Phase 27 demoted Pi to optional `--pi` profile; base stack runs without it.*
- [x] **CORE-02**: Pi adapter pattern established — single point of contact with pi-mono, version pinned to exact release, with documented upgrade procedure
  - *Outcome: Delivered v0.1. `@mariozechner/pi-coding-agent@0.66.1` pinned. pi-adapter.ts is sole importer.*
- [x] **CORE-03**: Sentinel Core (FastAPI) receives a Message Envelope via POST /message and returns an AI response envelope
  - *Outcome: Delivered v0.1. Verified 7/7 truths. Path B: LiteLLM-direct, no Pi in message route.*
- [x] **CORE-04**: LM Studio on Mac Mini confirmed as AI backend — Core can call it and receive a completion response
  - *Outcome: Delivered v0.1. LM Studio is one of many LiteLLM-supported providers.*
- [x] **CORE-05**: Token count is calculated before every LM Studio call; calls are rejected before submission if they would exceed context window
  - *Outcome: Delivered v0.1. token_guard.py, 422 on excess. Context window from model registry.*
- [x] **CORE-06**: `docker compose up` starts the full core system (Core + Pi harness) in a single command
  - *Outcome: Delivered v0.1. Phase 27: base stack is Core only; Pi added via `sentinel.sh --pi`.*
- [x] **CORE-07**: Docker Compose `include` directive pattern established in base compose — no module or interface uses `-f` flag stacking
  - *Outcome: Delivered v0.1. Pi /reset route added Phase 23. Pi profile gated Phase 27.*

### Memory Layer

- [x] **MEM-01**: Obsidian Local REST API plugin accessible from Core container; health check detects when Obsidian is not running and degrades gracefully
  - *Outcome: Delivered v0.2. ObsidianClient._safe_request() graceful-failure helper added Phase 25.*
- [x] **MEM-02**: Core retrieves user context file (`/core/users/{user_id}.md`) before building Pi prompt
  - *Outcome: Delivered v0.2. Path B: context retrieval now feeds LiteLLM-direct prompt.*
- [x] **MEM-03**: Core writes session summary to vault after each interaction
  - *Outcome: Delivered v0.2. BackgroundTasks pattern, write-selectivity threshold enforced.*
- [x] **MEM-04**: System demonstrates cross-session memory — a second conversation references a specific detail from a prior session
  - *Outcome: Delivered v0.2.*
- [x] **MEM-05**: Tiered retrieval architecture in place — hot tier (last N), warm tier (vault search), cold tier (archived)
  - *Outcome: Delivered v0.2/v0.4. search_vault() wired Phase 7.*
- [x] **MEM-06**: Write-selectivity policy defined — not every message exchange writes a session note; threshold documented
  - *Outcome: Delivered v0.2. Threshold documented and enforced.*
- [x] **MEM-07**: Token budget ceiling enforced for context injection
  - *Outcome: Delivered v0.2. 25% context budget pre-token-guard; prevents 422s on large profiles.*
- [x] **MEM-08**: Obsidian search interface abstracted behind a class
  - *Outcome: Delivered v0.4. search_vault() in production message pipeline (warm tier).*

### Interfaces

- [x] **IFACE-01**: Standard Message Envelope defined as a Pydantic v2 model
  - *Outcome: Delivered v0.3. MessageEnvelope with optional source/channel_id fields added Phase 25 (D-03).*
- [x] **IFACE-02**: Discord bot container operational using discord.py v2.7.x
  - *Outcome: Delivered v0.3. Restored Phase 21 after regression. /sentask renamed to /sen Phase 27.*
- [x] **IFACE-03**: Discord slash commands use deferred responses
  - *Outcome: Delivered v0.3. interaction.response.defer(thinking=True) first await in handler.*
- [x] **IFACE-04**: Discord multi-turn conversations use threads
  - *Outcome: Delivered v0.3. Thread ID persistence to ops/discord-threads.md (Phase 10/25).*
- [x] **IFACE-05**: Apple Messages bridge operational as feature-flagged tier-2 interface
  - *Outcome: Delivered v0.3. attributedBody NSKeyedArchiver decoding added Phase 25.*
- [x] **IFACE-06**: All non-health Core endpoints require `X-Sentinel-Key` header authentication
  - *Outcome: Delivered v0.3. APIKeyMiddleware. Module proxy forwards auth header (Phase 27-08).*

### AI Provider

- [x] **PROV-01**: All provider URLs and API keys configured via environment variables
  - *Outcome: Delivered v0.4. LiteLLM-direct, all config via env vars.*
- [x] **PROV-02**: At least two AI providers testable by changing only env vars
  - *Outcome: Delivered v0.4. LM Studio + Claude API (and any LiteLLM-supported provider).*
- [x] **PROV-03**: Pi client has error handling, retry logic (3 attempts, exponential backoff), and hard 30-second timeout
  - *Outcome: Delivered v0.4. retry_config.py centralized Phase 25 (DUP-01 closure). LMSTUDIO_TIMEOUT_S env var Phase 23.*
- [x] **PROV-04**: Model configuration registry maps each model name to its context window size and capabilities
  - *Outcome: Delivered v0.4. Hybrid live-fetch + seed.*
- [x] **PROV-05**: Provider fallback configured — when LM Studio unavailable, Core can route to Claude API
  - *Outcome: Delivered v0.4. ConnectError-only fallback via LiteLLM.*

### AI Security

- [x] **SEC-01**: Prompt injection attack surface documented and mitigations in place
  - *Outcome: Delivered v0.4. InjectionFilter restored Phase 21 after regression.*
- [x] **SEC-02**: Sensitive data does not leak through model responses
  - *Outcome: Delivered v0.4. OutputScanner restored Phase 21. SecondaryClassifier abstraction Phase 21.*
- [x] **SEC-03**: OWASP LLM Top 10 checklist reviewed and all applicable findings addressed
  - *Outcome: Delivered v0.4. owasp-llm-checklist.md restored Phase 24 (had been deleted).*
- [x] **SEC-04**: Jailbreak resistance baseline documented — automated pen test agent runs weekly
  - *Outcome: Delivered v0.40. Pentest agent compose include wired Phase 24.*

  - *Outcome: Delivered v0.4. Phase 26 added 4 additional tests for subcommand coverage.*
- [x] **2B-02**: asyncio.gather() parallel reads 5 self/ files
  - *Outcome: Delivered v0.4.*
- [x] **2B-03**: Thread ID persistence to ops/discord-threads.md
  - *Outcome: Delivered v0.4. Phase 26 added thread persistence test coverage.*
