# DRY Audit — 2026-04-27 — Sentinel of Mnemosyne

**Scope:** `modules/pathfinder/app/`, `sentinel-core/app/`, `interfaces/discord/bot.py`
**Reviewer stance:** adversarial; assumed every recent bug fix had a duplication tail

## Executive summary

The three Python services have crossed the threshold where copy-paste has produced **structurally-identical code that genuinely solves the same problem twice (or three times)**. Today's bug count is the symptom: `01c67ce` (resolve_model prefix) and `c36cc1e` (preserve HF namespaces) are the same logic error landing in two different files because the function was duplicated. `model_profiles.py` is byte-identical (modulo whitespace) across two services; the LiteLLM-prefix-strip helper has **three** implementations in sentinel-core alone (`main.py::_strip_litellm_prefix`, `model_registry.py:170-173` inline loop, `resolve_model.py` only handles `openai/`); the `_stop_for(profile)` helper is duplicated within a single package (`pathfinder/app/llm.py` ↔ `pathfinder/app/foundry.py`); and the `kwargs[...] = ...; stop = _stop_for(profile); if stop: kwargs["stop"]=stop` boilerplate fires before every `litellm.acompletion` call (11 sites pathfinder, 1 sentinel-core). Recommended actions are mostly **S/M-effort mechanical extractions**; the cross-container duplication of `model_profiles.py` is the only L-effort decision and the recommendation is to publish it as a shared package or mounted volume rather than continuing to maintain two divergent copies.

## HIGH — duplication that has caused at least one bug today

### H-1. LiteLLM provider-prefix handling has three implementations and only one is correct

**What is duplicated**
- `modules/pathfinder/app/resolve_model.py:41-43` — checks `startswith("openai/")` only; would mis-handle ollama/anthropic/HF model strings.
- `sentinel-core/app/main.py:48-63` — `_strip_litellm_prefix` strips a tuple of three prefixes (`openai/`, `ollama/`, `anthropic/`).
- `sentinel-core/app/services/model_registry.py:170-173` — yet another inline `for prefix in ("openai/", "ollama/", "anthropic/"): ...` loop.
- `sentinel-core/app/services/model_selector.py:44-59` (`_LITELLM_PROVIDER_PREFIXES`) — the **canonical** 14-prefix list; the only one that recognises HuggingFace/groq/gemini/etc.

**What it should look like** One function in a shared module:
```python
def strip_litellm_prefix(model_str: str) -> str:
    for p in _LITELLM_PROVIDER_PREFIXES:
        if model_str.startswith(p):
            return model_str[len(p):]
    return model_str

def ensure_litellm_prefix(model_str: str, default_provider: str = "openai/") -> str:
    if model_str.startswith(_LITELLM_PROVIDER_PREFIXES):
        return model_str
    return f"{default_provider}{model_str}"
```

**Effort** S — 4 sites, ~30 lines deleted.

**Bug-impact evidence** `01c67ce` and `c36cc1e` — same bug class, two services.

### H-2. `model_profiles.py` is duplicated byte-for-byte across two containers

**What is duplicated** `modules/pathfinder/app/model_profiles.py` (215 LoC) vs `sentinel-core/app/services/model_profiles.py` (221 LoC). `diff` reports only whitespace differences. The `FAMILY_PROFILES` dict, `_SUBSTRING_PATTERNS`, `_substring_match`, `get_profile`, the LM Studio `/api/v0` lookup, the cache, the cache lock — all identical.

**Why it duplicated** Cross-container constraint — Docker images don't share Python paths. `6b26f09` "feat(quick-260426-2x1): add model_profiles.py to pathfinder and sentinel-core".

**What it should look like**
- (a) Publish as a tiny package `sentinel-shared` and `pip install -e ../sentinel-shared` in both Dockerfiles.
- (b) Mount a shared volume in both compose services and add to `sys.path`.
- (c) Generate one from the other at image-build time (Dockerfile `COPY ../shared/...`).

The "duplicate it for now" decision is no longer defensible: when a new model family lands (Phi-4, Llama-4) **two files must be edited in lockstep** or the two services will disagree on stop sequences.

**Effort** M.

**Bug-impact evidence** Today's `6b95e86` ("missing import" fix) only landed in sentinel-core — same dual-ship hazard.

### H-3. Import-refactor sprawl in pathfinder routes (today's `9f449e1`)

**What is duplicated** Today's `9f449e1` added the *same* import line to **three** files because three routes independently call `resolve_model_profile`:
- `modules/pathfinder/app/routes/rule.py:42`
- `modules/pathfinder/app/routes/harvest.py:36`
- `modules/pathfinder/app/routes/npc.py:42`

Plus eight `(model, profile)` pair construction sites that each repeat:
```python
model = await resolve_model("structured")
profile = await resolve_model_profile("structured")
```

**What it should look like** Return both as a pair with a single await:
```python
@dataclass(frozen=True)
class ResolvedModel:
    model: str
    profile: ModelProfile
    api_base: str | None

async def resolve(task_kind: TaskKind, *, force_refresh: bool = False) -> ResolvedModel:
    ...
```
Routes become `r = await resolve("structured"); ... model=r.model, api_base=r.api_base, profile=r.profile`.

**Effort** S — 8 call sites, additive.

**Bug-impact evidence** `9f449e1` (today) is literally "fix the same missing import in three files."

### H-4. `_stop_for(profile)` helper duplicated *within the same package*

**What is duplicated**
- `modules/pathfinder/app/llm.py:28-38`
- `modules/pathfinder/app/foundry.py:20-25`

Plus the `if api_base: kwargs["api_base"] = api_base; stop = _stop_for(profile); if stop: kwargs["stop"] = stop` boilerplate before **11** `litellm.acompletion` calls in `llm.py` and once in `foundry.py:82-86`. Sentinel-core has the equivalent inside `LiteLLMProvider.complete` (`litellm_provider.py:70-83`), which is the **right** shape.

**What it should look like** Pathfinder should adopt the sentinel-core wrapper pattern:
```python
async def acompletion_with_profile(
    *, model: str, messages: list[dict], profile: ModelProfile | None,
    api_base: str | None = None, timeout: float = 60.0, **extra,
):
    kwargs: dict = {"model": model, "messages": messages, "timeout": timeout, **extra}
    if api_base: kwargs["api_base"] = api_base
    if profile and profile.stop_sequences: kwargs["stop"] = profile.stop_sequences
    return await litellm.acompletion(**kwargs)
```

**Effort** M — 12 mechanical replacements. Best done with H-3 in the same pass.

**Bug-impact evidence** `ea7da29` had to touch 11 sites by hand. Next profile field added (e.g. `temperature_preset`) would need the same sweep.

### M-4. Error-message stripping at the user boundary

- `interfaces/discord/bot.py:819` — `f"⚠ Rules query failed — {str(exc).splitlines()[0]}"` strips httpx MDN URL block.
- `interfaces/discord/bot.py:1144-1155` — `_pf_dispatch` exception handler does its own httpx → user-facing translation.

```python
def _user_safe_error(exc: Exception, *, prefix: str = "Module error") -> str:
    if isinstance(exc, httpx.HTTPStatusError):
        try: detail = exc.response.json().get("detail", exc.response.text)
        except Exception: detail = exc.response.text
        return f"{prefix} (HTTP {exc.response.status_code}): {detail}"
    if isinstance(exc, httpx.ConnectError): return "Cannot reach the Sentinel."
    if isinstance(exc, httpx.TimeoutException): return "The module took too long to respond."
    return f"{prefix} — {str(exc).splitlines()[0]}"
```

**Effort** S.

## Architectural smells (not pure duplication)

### A-1. `pathfinder/app/llm.py` is a god module (1053 LoC, 8 LLM-call functions)
Function-scope imports of `app.rules` (lines 546, 611, 683, 769, 1022) are a smell. Split into `llm/{npc,dialogue,harvest,rules,foundry}.py`, ~150 LoC each. **Effort:** M.

### A-2. `pathfinder/app/routes/npc.py` is 977 LoC, 9 endpoints
Split by verb. **Effort:** M.

### A-3. State threading via `app.state.lmstudio_stop_sequences` is fragile
`LiteLLMProvider` already accepts `stop=` per call; the provider should hold its profile so callers do `await provider.complete(messages)` and the provider knows its stop sequences. **Effort:** S.

### A-4. `model_selector.py` mixes module-level cache with instance-style discovery
Pick one ownership model. **Effort:** S.

### A-5. `interfaces/discord/bot.py` is 1594 LoC in a single file
Split into `bot/{client,routing,render,sentask}.py`. **Effort:** L.
