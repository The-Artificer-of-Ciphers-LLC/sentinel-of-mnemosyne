<objective>
Fix two sentinel-core code defects: narrow an overly broad bare except clause in the message route, and remove a dead method from the Pi adapter client.

Purpose: The bare `except Exception:` at message.py:149 masks Pi protocol errors (e.g., KeyError on a malformed response) as connectivity failures, silently triggering AI provider fallback when the real error is a protocol bug. The dead `send_prompt()` method in pi_adapter.py is unreachable production code that creates confusion about which method is canonical.

Output: Two modified files; full test suite still green; D-02 documented as pre-verified.
