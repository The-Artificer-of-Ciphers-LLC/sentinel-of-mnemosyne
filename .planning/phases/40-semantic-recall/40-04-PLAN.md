<objective>
Close the root-cause half of the Phase-40 UAT blocker AND tighten it per the SECOND round of cross-AI
review (40-REVIEWS.md, round 2 — Codex HIGH/not-converged). The first boot's "non-blocking startup index
rebuild" (D-06) called the FULL destructive `run_sweep` (walk → classify → embed → dedup → RELOCATE)
instead of an index-only routine. With models not yet loaded on first boot, classifications were degraded,
and the sweeper RELOCATED `sentinel/persona.md` → `learning/persona/`, crash-looping the next boot at
`composition.py:424`.

Round 1 moved the safety probe inside `run_sweep`; round 2 caught that this is still NOT safe as written:
(1) the guard was evaluated ONCE after lock acquisition, then later relocate/trash moves proceeded on that
stale decision; and (2) the proposed probe checked a generic "chat model resolvable" signal, but
`classify_note` uses the STRUCTURED model-selection path (`select_model("structured", ...)`), and
`select_model` returns a DEFAULT (rules 3/5) or a bare `loaded[0]` (rule 4) even when no model genuinely
scores for "structured" — so the probe could report "ready" on a degraded classifier and the guard is
bypassed at its source.

This plan:
1. Adds a NEW index-only routine `rebuild_embedding_index()` that walks the vault, (re)embeds bodies,
   and writes `ops/sweeps/embedding-index.json` via the existing `_emit_embedding_index` helper — with
   ZERO classify / relocate / trash / de-dup calls.
2. Rewires the startup rebuild in `composition.py` to call `rebuild_embedding_index()` instead of
   `run_sweep()`.
3. (ROUND-2 item A) Evaluates the RUNTIME "safe to mutate" probe IMMEDIATELY BEFORE EACH live destructive
   move (noise→trash, misplaced→relocate, dedup→trash), re-checking per move so that if readiness flips to
   False partway through a sweep, NO destructive move happens after that point — fail-closed.
4. (ROUND-2 item B) Binds the probe to the CLASSIFIER's REAL readiness via a new fail-closed
   `probe_classifier_model_ready()` that mirrors the structured-selection path `classify_note` uses and
   returns False when no genuinely-loaded model SCORES for the "structured" task kind (a defaulted/last-
   resort selection is NOT "ready"). The safe-to-mutate probe ANDs this with embedding-model readiness.
5. (ROUND-2 item C) Suppresses classification frontmatter write-back on unsafe runs too — degraded
   classifier output is not persisted ANYWHERE (not just no moves); the note is left byte-identical.
6. Specifies the degraded-index invariant: in an unsafe/degraded run, a changed note's entry is never
   rewritten with a new `content_hash` without a fresh vector (review concern 2 / MEM-05).
7. OWNS the sweeper-side exception handling: every destructive branch in `run_sweep` catches
   `ProtectedPathError` (from 40-05), records it in `report.errors`, and continues (review concern 3).
8. Wires the admin sweep entrypoint so a manual `POST /vault/sweep/start` passes the fail-closed runtime
   probe and cannot bypass the guard (review concern 4).

Purpose: a cold start must become semantically searchable WITHOUT ever moving a vault file, and a sweep
must never act destructively on degraded model output — at the point of EACH move, bound to the classifier's
actual readiness — OR abort wholesale on a single protected-path hit.
Output: `rebuild_embedding_index()`, a fail-closed `probe_classifier_model_ready()`, a per-move runtime
safe-to-mutate gate, frontmatter-write suppression on unsafe runs, per-branch ProtectedPathError handling,
the degraded-index invariant, the rewired startup path, admin-endpoint wiring, and regression tests.

This plan does NOT regress the 4 shipped success criteria (RetrievalStrategy seam, SemanticRecall reading
the sidecar, embedding_model mismatch skip, RRF k≈60 merge) — the index-write path (`_emit_embedding_index`)
is reused verbatim and only its degraded-run carry-forward semantics are made explicit.

CR-03 (incremental re-embed efficiency) remains explicitly DEFERRED — not planned here.
</objective>
