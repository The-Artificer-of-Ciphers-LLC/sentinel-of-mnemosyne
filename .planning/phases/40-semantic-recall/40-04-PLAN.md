<objective>
Close the root-cause half of the Phase-40 UAT blocker AND tighten it per cross-AI review (40-REVIEWS.md):
the first boot's "non-blocking startup index rebuild" (D-06) called the FULL destructive `run_sweep`
(walk → classify → embed → dedup → RELOCATE) instead of an index-only routine. With models not yet loaded
on first boot, classifications were degraded, and the sweeper RELOCATED `sentinel/persona.md` →
`learning/persona/`, crash-looping the next boot at `composition.py:424`.

This plan:
1. Adds a NEW index-only routine `rebuild_embedding_index()` that walks the vault, (re)embeds bodies,
   and writes `ops/sweeps/embedding-index.json` via the existing `_emit_embedding_index` helper — with
   ZERO classify / relocate / trash / de-dup calls.
2. Rewires the startup rebuild in `composition.py` to call `rebuild_embedding_index()` instead of
   `run_sweep()`.
3. Replaces the narrow boot-time `model_loaded` boolean with a RUNTIME "safe to mutate" probe evaluated
   INSIDE `run_sweep` immediately before any destructive step, broadened from "embeddings available" to
   "every prerequisite for trustworthy destructive classification is ready" — embedding model AND
   classifier/chat model (review concern 1).
4. Specifies the degraded-index invariant: in an unsafe/degraded run, a changed note's entry is never
   rewritten with a new `content_hash` without a fresh vector (review concern 2 / MEM-05).
5. OWNS the sweeper-side exception handling: every destructive branch in `run_sweep` catches
   `ProtectedPathError` (from 40-05), records it in `report.errors`, and continues (review concern 3) —
   this closes the cross-plan ownership hole the reviewers flagged.
6. Wires the admin sweep entrypoint so a manual `POST /vault/sweep/start` passes the runtime probe and
   cannot bypass the guard (review concern 4).

Purpose: a cold start must become semantically searchable WITHOUT ever moving a vault file, and a sweep
must never act destructively on degraded model output OR abort wholesale on a single protected-path hit.
Output: `rebuild_embedding_index()`, a runtime `_safe_to_mutate` probe, per-branch ProtectedPathError
handling, degraded-index invariant, rewired startup path, admin-endpoint wiring, and regression tests.

This plan does NOT regress the 4 shipped success criteria (RetrievalStrategy seam, SemanticRecall reading
the sidecar, embedding_model mismatch skip, RRF k≈60 merge) — the index-write path (`_emit_embedding_index`)
is reused verbatim and only its degraded-run carry-forward semantics are made explicit.

CR-03 (incremental re-embed efficiency) remains explicitly DEFERRED — not planned here.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| boot process → vault (REST) | The non-blocking startup task mutates the vault via the Vault seam. Before this fix it could RELOCATE/TRASH operator files on degraded classifications. |
