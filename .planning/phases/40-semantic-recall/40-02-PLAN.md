MEM-05 ("no per-NOTE HTTP at query time").

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| index JSON (at rest) → SemanticRecall | The sweeper-written `embedding-index.json` is read back and parsed; its bytes derive from user note content |
| embedding_b64 string → numpy decode | Per-entry base64 is decoded into a float vector before the cosine scan |
| user message → embed_fn | The full raw message content is embedded as the query vector |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-40-05 | Tampering | `_load_index_if_stale` parsing index JSON | mitigate | `json.loads` in try/except → `{}` on any failure; per-entry `.get()` access (no KeyError); malformed/injected JSON degrades to keyword-only, never crashes the request |
| T-40-06 | Tampering | decoding `embedding_b64` / oversized vector | mitigate | Delegate to `decode_embedding` (returns `[]` on bad base64); skip zero-length vectors (Pitfall 4); `np.asarray` bounds the vector — a malformed entry is dropped, not fatal |
| T-40-07 | Information disclosure | path keys in the index surfacing ops/self/trash content into recall | mitigate | SemanticRecall applies `config.exclude_prefixes` to index keys (same fence KeywordRecall uses) so `ops/`, `_trash/`, `self/` never enter the RRF pool |
| T-40-08 | Spoofing / cross-model results | comparing an old-model vector against a new-model query | mitigate | Exact-string `embedding_model == active_model` (no normalization, no `startswith`) per D-12; missing/empty → skip (D-13); all-mismatch → `[]` (D-14) — geometrically-meaningless comparisons are excluded |
| T-40-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages this phase (RESEARCH Package Legitimacy Audit: none) |
</threat_model>
