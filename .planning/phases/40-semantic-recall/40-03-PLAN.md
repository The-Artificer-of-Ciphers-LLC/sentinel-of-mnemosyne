<objective>
Wire the producer (Plan 01 sweeper index) and the consumer (Plan 02 SemanticRecall) together
at the composition root, add the on-demand rebuild trigger (D-06), and prove the full hybrid
recall path end-to-end (MEM-03, MEM-04, MEM-05).

Composition (RESEARCH Open Question 5, PATTERNS composition.py section): construct
`SemanticRecall(vault, embed_fn=embeddings.embed, active_model=settings.embedding_model,
config=...)` and inject it into `Recall(..., semantic_strategy=semantic)` inside the
`if recall is None:` guard. `active_model` MUST be `settings.embedding_model` (NO `openai/`
prefix) so it matches the `embedding_model` the sweeper writes (Plan 01) — using
`Embeddings._model` (which has the prefix) would make every model-match fail (D-12).

On-demand rebuild (D-06): the index is emitted inside `run_sweep` (Plan 01), and the
existing admin-gated `POST /vault/sweep/start` route already invokes `run_sweep` — so the
on-demand rebuild trigger is satisfied by the existing endpoint with NO new authz surface.
Add a non-blocking startup pass (`asyncio.create_task`) so a cold start with no prior index
becomes searchable without operator action.

Purpose: Close the write↔read gap so the sweeper's embeddings become live retrieval data.
Output: wired `composition.py` (embeddings-before-recall ordering fix + SemanticRecall
injection + startup rebuild task), composition wiring test, end-to-end integration test.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client → POST /vault/sweep/start | The on-demand index rebuild trigger is reachable over HTTP and runs a full (potentially expensive) sweep |
| startup → background sweep task | A non-blocking task runs at process start with the sweep's vault/embedder authority |
| composition → active_model wiring | The model-match correctness hinges on the exact string injected as active_model |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-40-09 | Elevation of privilege / unauthorized trigger | on-demand rebuild endpoint | mitigate | Reuse the EXISTING `POST /vault/sweep/start` route which already enforces `_is_admin_route` (403 for non-admin) — no new unauthenticated surface is introduced; the rebuild rides the already-gated route |
| T-40-10 | Denial of Service | repeated admin-triggered full re-embeds | accept | The trigger is admin-gated (T-40-09); the existing sweep status guard (`get_status`/running) prevents overlapping sweeps; cost is bounded at personal-vault scale. Steady state is incremental (Plan 01 content-hash) |
| T-40-11 | Tampering / correctness | wrong active_model string (prefixed vs unprefixed) | mitigate | Wiring asserts `active_model == settings.embedding_model` (no `openai/` prefix) in a composition test; a regression flips model-match off → caught by the all-mismatch + end-to-end tests |
| T-40-12 | Denial of Service | startup rebuild task crashing or blocking startup | mitigate | Task is non-blocking (`create_task`) and failure-tolerant (warn-and-continue, reusing the lifespan's Obsidian-unavailable graceful path) — a rebuild failure never blocks or crashes startup |
| T-40-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages this phase (RESEARCH Package Legitimacy Audit: none) |
</threat_model>
