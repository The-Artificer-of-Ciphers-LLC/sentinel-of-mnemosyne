<objective>
Promote the `.json`→`.md` index-path fallback from a buried one-liner inside the 40-06 human checkpoint
into a real, testable, atomic change (cross-AI review concern 10 / 40-REVIEWS.md), and tighten it per the
SECOND round of review (item D): both reviewers flagged that the round-1 plan still permitted a weak
alternative — "duplicate literals + equality-test" — which is NOT single-source. This plan now MANDATES a
single physical constant: `RecallConfig.index_path` DERIVES FROM (imports) `EMBEDDING_INDEX_PATH` so the two
sides cannot diverge by construction; the equality test merely guards the derivation. The extension check is
also made CASE-INSENSITIVE per round-2.

The fallback is a production code change touching the index path in TWO modules
(`EMBEDDING_INDEX_PATH` in vault_sweeper.py and `RecallConfig.index_path` in recall.py); if the live
Obsidian REST API rejects the `.json` extension, both must change ATOMICALLY or the sweeper writes one path
while recall reads another and recall silently returns nothing. A single derived constant makes that
impossible.

This plan:
1. Makes the index path a SINGLE PHYSICAL source of truth — `RecallConfig.index_path` imports and uses
   `EMBEDDING_INDEX_PATH` as its field default rather than re-stating the literal. The round-1
   "duplicate literals + assertion" alternative is REMOVED — derivation is required, not optional. A test
   asserts the two are equal as a guard against an accidental future literal.
2. Makes the index body encode/decode CASE-INSENSITIVELY EXTENSION-AWARE so the same code round-trips for
   both `.json` (raw JSON body) and `.md` (JSON wrapped in a markdown fenced code block, which the Obsidian
   REST API accepts as a note). The writer (`_emit_embedding_index`) fences when
   `path.lower().endswith(".md")`; the reader (`SemanticRecall` index load) strips the fence before
   `json.loads`. For `.json`, both are pass-through.
3. Adds a regression test proving round-trip on BOTH extensions: write via the sweeper helper, read via the
   recall loader, assert the parsed index is byte-equal — once with a `.json` path and once with a `.md`
   path — so flipping the single constant is a one-line change that is already proven to work end-to-end.

This makes 40-06's live UAT Test 3 a CONFIRMATION step (round-trip the path 40-07 selected) rather than a
risky live code change buried in a checkpoint. 40-06 depends on this plan.

Purpose: the `.json`/`.md` decision becomes a pre-tested, atomic SINGLE-constant flip — there is no window,
and no representation, where the writer and reader can disagree on the index path.
Output: a single derived index-path constant, case-insensitive extension-aware fenced-JSON encode/decode,
and a both-extensions round-trip regression test.

This plan does NOT change the DEFAULT path (`.json` stays the default unless the live API rejects it in
40-06); it makes the `.md` fallback a tested, one-flip change. CR-03 remains deferred.
</objective>
