<objective>
Promote the `.json`→`.md` index-path fallback from a buried one-liner inside the 40-06 human checkpoint
into a real, testable, atomic change (cross-AI review concern 10 / 40-REVIEWS.md). Two reviewers flagged
that this fallback is a production code change touching TWO constants in TWO modules
(`EMBEDDING_INDEX_PATH` in vault_sweeper.py and `RecallConfig.index_path` in recall.py); if the live
Obsidian REST API rejects the `.json` extension, both must change ATOMICALLY or the sweeper writes one path
while recall reads another and recall silently returns nothing.

This plan:
1. Makes the index path a SINGLE source of truth — `RecallConfig.index_path` references the SAME constant
   as `EMBEDDING_INDEX_PATH` (import it or assert equality) so the two can never silently diverge. A test
   asserts they are equal.
2. Makes the index body encode/decode EXTENSION-AWARE so the same code round-trips for both `.json` (raw
   JSON body) and `.md` (JSON wrapped in a markdown fenced code block, which the Obsidian REST API accepts
   as a note). The writer (`_emit_embedding_index`) fences when the active path ends in `.md`; the reader
   (`SemanticRecall` index load) strips the fence before `json.loads`. For `.json`, both are pass-through.
3. Adds a regression test proving round-trip on BOTH extensions: write via the sweeper helper, read via the
   recall loader, assert the parsed index is byte-equal — once with a `.json` path and once with a `.md`
   path — so flipping the constant is a one-line change that is already proven to work end-to-end.

This makes 40-06's live UAT Test 3 a CONFIRMATION step (round-trip the path 40-07 selected) rather than a
risky live code change buried in a checkpoint. 40-06 depends on this plan.

Purpose: the `.json`/`.md` decision becomes a pre-tested, atomic dual-constant flip — no window where the
writer and reader disagree on the index path.
Output: shared index-path constant, extension-aware fenced-JSON encode/decode, and a both-extensions
round-trip regression test.

This plan does NOT change the DEFAULT path (`.json` stays the default unless the live API rejects it in
40-06); it makes the `.md` fallback a tested, one-flip change. CR-03 remains deferred.
</objective>
