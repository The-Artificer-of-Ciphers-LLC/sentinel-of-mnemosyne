<objective>
Close the protection half of the Phase-40 UAT blocker: the sweeper RELOCATED the operator-critical
`sentinel/persona.md` because nothing in the vault treated `sentinel/` as never-movable. `sentinel/` is
NOT in `sweep_skip_prefixes` (config.py lines 102-117) — and even if it were, skip-prefix only governs
whether a note is PROCESSED, not whether it may be MOVED. The persona crash-looped boot at
`composition.py:424` (`'sentinel/persona.md missing from Vault'`).

This plan adds a Vault-seam-level protected-namespace guard so that operator-critical paths can NEVER be
relocated or trashed — by ANY caller (startup rebuild, admin sweep, force_reclassify sweep, or a future
code path). The guard lives on `ObsidianVault.relocate` and `ObsidianVault.move_to_trash` — the two
mutation primitives that physically move files (vault.py:395-488) — so it is enforced regardless of which
decision logic invoked them.

Purpose: defense-in-depth. Even if 40-04's startup/degradation guards were bypassed by a future
regression, a protected file physically cannot be relocated or trashed.
Output: `PROTECTED_NAMESPACES` set + `is_protected_path()` predicate + `ProtectedPathError`, enforced
inside `relocate`/`move_to_trash`, defaulting to protect `sentinel/`, env-overridable via
`protected_namespaces`.
</objective>
