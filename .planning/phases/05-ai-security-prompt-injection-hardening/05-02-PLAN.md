<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| app.state → message.py | Security services must be accessed from app.state, not re-instantiated per request |
| message.py pipeline order | Framing must apply AFTER truncation (Pitfall 2); output scan must apply BEFORE BackgroundTasks write |
| Blocked response → Obsidian | Leaked content must NOT be written verbatim to Obsidian incident log |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-07 | Tampering | message.py pipeline ordering | mitigate | wrap_context() called with safe_context (post-truncation), not raw_context. Enforced by code review of injection_filter call site. |
| T-05-08 | Information Disclosure | _log_leak_incident writing blocked content to Obsidian | mitigate | Log writes block_reason only, not the blocked response content. Wording: "Original content withheld from this log for safety." |
| T-05-09 | Denial of Service | OutputScanner.scan blocks entire response pipeline | mitigate | scan() is async, non-blocking; timeout enforced inside OutputScanner (2s). HTTPException only raised on confirmed LEAK (SAFE + timeout both return True). |
| T-05-10 | Spoofing | Mock injection_filter in tests allows bypass | accept | Test mocks are test-only; production lifespan always instantiates real InjectionFilter. No test-only bypass path in production code. |
</threat_model>
