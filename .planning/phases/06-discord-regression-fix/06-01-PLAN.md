# Plan 06-01: Uncomment Discord Include

## Goal
Restore the Discord bot container to `docker-compose.yml` by uncommenting the single line that was removed in commit `bf7a704`.

## Context
During Phase 4 work (commit `bf7a704`), the Discord interface include line was commented out. The bot code, Dockerfile, and `interfaces/discord/compose.yml` are unchanged and still correct from Phase 3. Only the root `docker-compose.yml` needs a one-character edit.

Per D-01: only the include line in `docker-compose.yml` changes. Per D-02: no other file changes.

## Tasks

### Task 1: Uncomment the Discord include line
**File:** `docker-compose.yml`
**Action:** modify

**Details:**

Current state of `docker-compose.yml` (lines 10-12):
```
# Future modules (uncomment when phase is ready):
#   - path: interfaces/discord/compose.yml
#   - path: modules/pathfinder/compose.yml
```

Move the discord entry into the active `include:` block (above the `# Future modules` comment), and remove it from the comment block. The result should be:

```yaml
include:
  - path: sentinel-core/compose.yml
  - path: pi-harness/compose.yml
  - path: security/pentest-agent/compose.yml
  - path: interfaces/discord/compose.yml

# Future modules (uncomment when phase is ready):
#   - path: modules/pathfinder/compose.yml
```

The `modules/pathfinder/compose.yml` line stays commented — it is not part of this phase.

## Verification

```bash
# 1. Validate compose config parses correctly and discord service is present
docker compose config --services
# Expected output includes: discord (or whatever the service name is in interfaces/discord/compose.yml)

# 2. Confirm no syntax errors
docker compose config > /dev/null && echo "OK"
```

## Files Modified
- `docker-compose.yml` — moved discord include from commented block into active include list
