<objective>
Fix the asyncio_mode gap so all async tests actually execute instead of silently returning coroutine objects.

Purpose: Every async test in this codebase has been a false-positive since it was written. pytest-asyncio requires asyncio_mode = "auto" to be set via the ini options system. sentinel-core/pyproject.toml already has this correctly set. interfaces/discord has NO pyproject.toml at all — its conftest.py uses config.option.asyncio_mode = "auto" which is unreliable and NOT the same as [tool.pytest.ini_options]. This plan creates the missing pyproject.toml for the discord interface and then runs both suites to confirm tests actually execute.

Output: interfaces/discord/pyproject.toml with asyncio_mode = "auto" in [tool.pytest.ini_options]. Verified: both pytest suites run and pass (not just collect).
</objective>
