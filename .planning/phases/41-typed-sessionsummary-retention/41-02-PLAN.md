<objective>
Retype the one bounded Vault-seam method this phase reopens: `get_recent_sessions`.
Across the `Vault` Protocol, `ObsidianVault`, and `FakeVault` (plus the alias) — in
lockstep — change the return from `list[str]` to `list[SessionSummary]` and the
parameter from `limit: int = 3` to `policy: RetentionPolicy`. Add the adapter-edge
parser that turns raw session markdown into a `SessionSummary`, and move the inline
today+yesterday window into `policy.hot_window_days`.

Purpose: MEM-08 (typed session values at the Vault edge), MEM-06 (the window/limit
become `RetentionPolicy` fields, no inline magic numbers). MEM-07 reachability is
verified-already (carrier note outside `ops/`), so no carrier code is built here —
the typed hot-tier read is the in-scope change.

Output: a typed, policy-driven `get_recent_sessions` on all three impls, with the
adapter-edge parser and updated adapter tests.

Scope guard (D-07 / ADR-0005): ONLY `get_recent_sessions` changes on the Vault
surface. Do NOT retype `find()`, `list_under()`, `read_note()`, or any other
Protocol method. The `f"{user_id}-" in filename` substring rule and the
`_safe_request(..., [], "get_recent_sessions")` graceful-degrade envelope are
preserved exactly.
</objective>
