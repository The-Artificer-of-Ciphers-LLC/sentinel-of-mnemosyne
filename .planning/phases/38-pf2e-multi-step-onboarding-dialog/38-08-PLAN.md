SPEC Acceptance Criteria checkboxes (verbatim from 38-SPEC.md):
1. `:pf player start` with no args creates a new thread and posts the character-name prompt as its first message
2. A plain-text reply in a draft-bearing thread updates the draft and posts the next prompt; the AI is not invoked
3. Bot restart between answers preserves the draft; resuming with the next answer completes correctly
4. After all three answers, `mnemosyne/pf2e/players/{slug}/profile.md` exists with `onboarded: true` and matches the pipe-syntax outcome byte-for-byte
5. Mid-dialog `:pf player note|ask|npc|recall|todo|style|canonize` returns the rejection template and does NOT mutate the vault
6. `:pf player cancel` with a draft deletes the draft file and replies with the cancel-acknowledgement message
7. `:pf player cancel` with no draft replies `No onboarding dialog in progress.` and does nothing
8. `:pf player start` re-issued in a thread with an in-flight draft re-posts the prompt for the current `step` and does NOT reset prior answers
9. `:pf player start a | b | c` (pipe syntax) regression: produces the same `profile.md` it did before this phase, with no thread created
10. Wave-0 RED tests exist for every requirement above and were written and committed BEFORE the implementation — VERIFIED MANUALLY in 38-09 Task 3 via `git log --diff-filter=A --format='%H %s' -- 'interfaces/discord/tests/test_pathfinder_player_dialog.py' 'interfaces/discord/tests/test_dialog_router.py'`
