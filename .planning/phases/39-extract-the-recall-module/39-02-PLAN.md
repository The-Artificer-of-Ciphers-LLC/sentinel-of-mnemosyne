<objective>
Trim `MessageProcessor` to delegate hot-tier and warm-tier assembly to `Recall.assemble()` (the module created in Plan 01), and wire `Recall` through the DI graph so both the message path and (in Plan 03) the `/context` route share one assembler.

This is the core of MEM-01: after this plan, `Recall.assemble()` is the sole entry point for hot+warm assembly on the message path — `MessageProcessor` contains no inline tier-building logic. `MessageProcessor` retains only the presentation/defense concerns it owns per D-04: the Sentinel persona read + `messages[0]` swap, `injection_filter.wrap_context()`, the "Understood." pairs, `_format_search_results`, and the final `TokenBudget.check()`. The constants that moved to `RecallConfig` in Plan 01 are deleted from `message_processing.py` (MEM-02), with `SEARCH_SCORE_THRESHOLD` re-exported so the existing `app/routes/message.py` import keeps working.

The behavior-preserving constraint is paramount: the kept `test_message_processor.py` and through-`/message` `test_message.py` suites are the regression net and MUST stay green.

Purpose: Single retrieval entry point on the message path (MEM-01); inline constants consolidated (MEM-02); Recall injected via the existing constructor-injection seam (Claude's discretion, D-04/D-06).
Output: trimmed `message_processing.py`, DI wiring in `composition.py` + `state.py`.
</objective>
