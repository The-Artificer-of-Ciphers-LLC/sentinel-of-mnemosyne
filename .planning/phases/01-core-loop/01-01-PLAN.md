## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-00-01 | Tampering | `.env.example` | accept | `.env.example` is a template with no secrets; only documents variable names. `.env` is gitignored. Low risk. |
| T-1-00-02 | Denial of Service | `depends_on: service_started` | accept | Using `service_started` (not `service_healthy`) allows Core to start before Pi is ready — intentional per graceful degradation locked decision. Core handles Pi unavailability via 503, not crash. |

Note: Phase 1 primary threats (T-1-01 token guard DoS, T-1-02 Pi subprocess injection, T-1-03 LM Studio open on local network) are mitigated in Plan 03 where the actual handlers are implemented.
